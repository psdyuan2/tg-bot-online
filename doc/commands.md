# 支持的 Telegram 命令说明

本文档列出当前**三机器人**（管理端 / 商户通知端 / 分红端）**已实现且可用**的全部命令。命令名均为**英文**，符合 Telegram 标准 Bot Command 格式。

---

## 管理机器人（Admin Bot）

在内部管理群使用，需配置环境变量 `ADMIN_BOT_TOKEN`。**不进行用户 ID 白名单校验**（任何能向该机器人发消息的用户均可触发管理命令，请务必通过群权限与 Token 保管控制访问面）。

| 命令 | 语法 | 说明 |
| --------- | ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `/uset` | `/uset <rate>` | 设置**实际 U 价**（写入 `system_config` 的 `U_RATE`）。**商户 U 价**在系统内为「实际 U 价 + 0.5」，用于商户侧回 U、quote、部分报表。示例：`/uset 101` |
| `/set_payout_fit` | `/set_payout_fit <rate>` | 设置**结算入账服务费率**（写入 `SETTLE_FEE_RATE`），默认 **6.5%**（可用 `6.5` 或 `0.065`）。账本入账净额 = 结算总额 × (1 − 该费率)。 |
| `/settle` | `/settle <merchant_identifier> <amount>` | **结算入账**：按当前配置的结算费率扣佣后增加商户余额并写流水；按「净入账 × 分红率」折算为 USDT 后累加到该商户的**分红余额**（除以**当次商户 U 价**）。回复与通知中含**当前 UTC 时间（精确到秒）**。商户标识同下。 |
| `/report` | `/report <merchant_identifier>` | **日终对账**：汇总**当日**（UTC 日历日）该商户结算与代付；**管理群**收到英文 Settlement 模板（含固定 **4%** 网关/Sky 展示行；**USDT 换算用实际 U 价**）；**商户群**收到 INR 中文模板（**结余换汇用商户 U 价**）。执行后该商户**结算余额 `balance` 清零**（当天余额视为已全部回 U）。 |

## 通知机器人（Notify Bot）

部署在商户群，需配置 `NOTIFY_BOT_TOKEN`。群内命令通过**当前群的 `chat_id`** 关联 `merchants`；首次使用前请在群内执行 **`/add_id`**。

| 命令 | 语法 | 说明 |
| --------- | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/add_id` | `/add_id` | 在**群组 / 超级群**内生成本群**短码**（`merchant_code`），必要时创建商户行。 |
| `/see_id` | `/see_id` | 显示当前群短码。 |
| `/u` | `/u` | 回复「今日u价」+ **商户 U 价**（实际 U + 0.5）。 |
| `/payout` | `/payout <amount>` | **代付扣款**：扣款额 = 申请金额 × **1.01**（含 **1%** 服务佣金从余额扣）；展示含**当前 UTC 时间（秒）**、银行手续费 **1.5%**、实际到账 **×0.985**、扣款后余额、**可回 U**（余额 ÷ **商户 U 价**）。同一商户群 **3 秒内**不可连续两次 `/payout`。 |
| `/quote` | `/quote <amount>` | 按**商户 U 价**估算 USDT，不修改余额。 |

---

## 分红机器人（Benefit Bot）

部署在商户群，需配置 `BENEFIT_BOT_TOKEN`。按群内 `tg_chat_id` 绑定商户。

| 命令 | 语法 | 说明 |
| --------- | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/set_fit` | `/set_fit <rate>` | 设置全局**分红率**（`DIVIDEND_RATE`，如 `1` 表示 1%）。在每次 `/settle` 成功后，将**净入账 × 分红率**对应本币金额除以**当时商户 U 价**得到 USDT，**累加**到该商户 `benefit_balance`。 |
| `/balance` | `/balance` | 在群内查询当前商户**分红累计（USDT）**。 |
| `/clear` | `/clear` | **结清分红**：将当前群商户的 `benefit_balance` 置零。 |

---

## 金额与格式

- 金额可为整数或小数；支持千分位逗号，例如 `1,234,567.89`。
- 结算/代付等本币金额在系统内以「分」为整数存储；USDT 与分红余额使用 `Decimal` 精度存储与展示。

---

## 部署与数据前提

- 数据库为 **SQLite**，启动前执行：`alembic upgrade head`。
- 需配置三个 Bot Token：`ADMIN_BOT_TOKEN`、`NOTIFY_BOT_TOKEN`、`BENEFIT_BOT_TOKEN`。
- 使用 `/settle`、`/report` 前，管理端应能解析商户（短码 / 商户名 / 数字 id）。

---

## 环境变量（运维对照）

| 变量 | 用途 |
| ------------------ | ---------------------------------------------------------------------------------------------------------------- |
| `ADMIN_BOT_TOKEN` | 管理机器人 Token |
| `NOTIFY_BOT_TOKEN` | 通知机器人 Token |
| `BENEFIT_BOT_TOKEN` | 分红机器人 Token |
| `DATABASE_URL` | 异步 SQLite 连接串 |
| `DEFAULT_U_RATE` | 首次读取且库中无配置时的默认**实际** U 价 |
| `DEFAULT_SETTLE_FEE_RATE` | 默认结算入账服务费率（如 `0.065`） |
| `DEFAULT_DIVIDEND_RATE` | 默认分红率（如 `0.01`） |
